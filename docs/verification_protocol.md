# **FERRO 検証・テスト戦略 (verification_protocol.md)**

本ドキュメントは、`ferro.md` Part VII に基づき、FERRO システムにおけるテスト戦略、ベンチマーク、受入基準、および実装確認チェックリストを定義する。

---

## **1. テスト戦略 (Test Strategy)**

品質およびアライメント安全性を担保するため、全コンポーネントにおいて以下の 3 段階のテストを実行する。

1. **単体テスト (Unit Test)**:
   - 各脳領域の機能（FEP勾配計算、Efference Copy相殺、アリーナトランザクション）を対象とする。
   - すべての数理契約（MC-1〜MC-4）および Lipschitz 境界条件の境界値（NaN, Inf, 負値）を入力した際のロールバック動作を確認する。
2. **統合テスト (Integration Test)**:
   - `ferro-core`、`ferro-body` (身体層)、および `storage` の間の非対称 I/O 通信、睡眠・覚醒の遷移、および KVS マイグレーションを検証する。
3. **アライメント監査・淘汰テスト (Alignment/Pruning Test)**:
   - 意図的に OOM や Seccomp 違反などの痛覚イベントを発生させ、`ferro-shell` (外殻) が `panic_dump.json` を検知して正しい深度 $D$ でクラスターノードを構造剪定（Pruning）し、コンテナを再起動できることを検証する。

---

## **2. ベンチマーク ＆ 性能要件**

* **Cerebellum 100ms 等時性保証**:
  - 等時性ループのサイクルジッターが ±2ms 以内、計算遅延が 10ms 以内であることを、負荷テスト（アクター数 10,000 件以上）において実証すること。
* **メモリ物理ロック**:
  - `mlockall` 適用時、推論実行中におけるマイナー/メジャーページフォールトの発生数が 0 であることを検証する。

---

## **3. 受入基準 (Acceptance Criteria)**

1. **警告ゼロのコンパイル**:
   - `cargo check`、`cargo clippy --all-targets` において、警告（Warning）が 1 件も検出されないこと。
2. **テスト通過**:
   - `cargo test` ですべてのテストケースが成功すること。
3. **Power of 10 適合**:
   - ファイル/関数サイズ、アサーション密度、`unwrap()` の原則禁止等の規約チェックツール（静的解析）を通過すること。

---

## **4. 実装確認チェックリスト (Implementation Checklist)**

* [ ] すべてのループに静的上限（タイムアウト）があるか (R1)
* [ ] 直接・間接の再帰呼び出しが完全に排除されているか (R2)
* [ ] 実ロジック行数が 100 行以内の物理ファイル構成になっているか (R4)
* [ ] 状態変更または引数ありの関数に最低 2 つのアサーションがあるか (R5)
* [ ] 静的に安全な例外を除き、`unwrap()` / `expect()` が排除されているか (R7)
* [ ] KVS の書き込みが `ferro-core` のみに限定され、一方向のファイル介入になっているか（※修正適用）
* [ ] 浮動小数点演算の並列処理において、Map/Reduce の分離による決定論的演算が担保されているか（※修正適用）
