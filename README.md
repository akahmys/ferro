# **FERRO: Hierarchical Active Inference System (Version 1.0)**

本アーキテクチャは、自由エネルギー原理（Free Energy Principle: FEP）に基づく多重マルコフブランケット（Markov Blanket）の数理モデルを応用し、隔離されたローカル環境下において自律的な資源制御および能動的推論（Active Inference）による自己コード改変を自律的に遂行する階層型ソフトウェアアーキテクチャの提案である。

## **1. システムトポロジー（極小感覚・運動分岐 ＆ 自己受容随伴発射 ＆ コンテナ隔離型マルコフブランケット）**

本システムは、情報理論的境界であるマルコフブランケットによって隔離された3つのソフトウェアレイヤーが、ファイルI/Oおよび非同期メッセージチャネルを介して非対称に相互作用する構造をとる。自律変異パッチの安全検証およびコアロジックの実稼働（実行）は、ホスト環境から完全に隔離された使い捨て型（Ephemeral）Dockerコンテナサンドボックス（`ferro-sandbox` / `ferro-core-runtime`）の内部でクローズに執行される。

1. **環境レイヤー（ferro-env）**:  
   ホストOSのシステムメトリクス（CPU使用率、メモリ占有量、ディスクI/Oスループット）の測定、およびマルチモーダルな入力ソース（ファイル監視、システムログストリーム、出力テキスト/音声バッファ）の供給を担う、外的な物理シミュレーション領域。  
2. **外殻統治レイヤー（ferro-shell）**:  
   `ferro-core-runtime` コンテナのプロセスの健全性を常時監視・制限するハイパーバイザ。ZPD（発達最近接領域）アルゴリズムに基づいた入力刺激の複雑度制御、提案されたパッチコード（PatchTicket）のDockerコンテナによる動的隔離検証、およびアライメント規則違反クラスターに対する「自動トポロジー剪定（Pruning）」と初期再デプロイを一貫して制御するマルチエージェント型統治システム。  
3. **コアロジックレイヤー（ferro-core）**:  
   予測誤差（自由エネルギー）の最小化を目的関数とし、自律的な知識グラフ構造の自己組織化、およびソースコードの自己変異（変異仮説の生成）を実行する、Dockerコンテナ（`ferro-core-runtime`）内に隔離されたコアバイナリ。

## **2. コアロジックレイヤー (ferro-core) における 5+1 階層構造の機能定義**

### **⓪ 極小感覚・運動アクターレイヤー (Sensory & Motor Organs / Micro-receptors & Actuators)**
* **体性感覚（内受容感覚：Interoceptors / 脳幹へ直結）**:  
  * **CPU温度監視アクター (skin/cpu_temp.rs)**: CPU温度を秒単位でサンプリング。前回測定値からの変動を検出した時のみ `InteroceptiveSignal::CpuTemp(f32)` を脳幹へ送信。  
  * **メモリ占有量監視アクター (skin/ram_free.rs)**: 空き物理メモリ量を監視。変動検知時に `InteroceptiveSignal::RamFree(u64)` を脳幹へプッシュ。  
  * **ストレージ負荷監視アクター (skin/disk_io.rs)**: ディスクI/Oスループットを計測。変動検知時に `InteroceptiveSignal::DiskIo(f64)` を脳幹へプッシュ。  
  * **例外発生率監視アクター (skin/process_error.rs)**: プロセス内で捕捉された例外の発生頻度を積算・監視。変動検知時に `InteroceptiveSignal::ProcessError(u32)` を脳幹へプッシュ。  
* **特殊感覚（外受容感覚：Exteroceptors / 小脳へ直結）**:  
  * **視覚変化検出アクター (eye/frame_delta.rs)**: `ferro-env` より提供されるフレームの変化率のみを算定。変化率が定義された閾値を超える場合のみ `SensorySignal::FrameDelta(f64)` を小脳へ送信。  
  * **視覚特徴抽出アクター (eye/image_embedding.rs)**: 特徴ベクトル `SensorySignal::ImageEmbedding(Vec<f32>)` を小脳へ送信。  
  * **音響特徴抽出アクター (ear/mfcc.rs)**: 音声波形からMFCCを抽出。`SensorySignal::Mfcc(Vec<f32>)` を小脳へ送信。  
  * **音響デコードアクター (ear/speech_token.rs)**: 音声情報を文字列配列に変換。`SensorySignal::SpeechToken(Vec<String>)` を小脳へ送信。  
  * **開発ログ監視アクター (dev_log/file_write.rs)**: ログファイルの更新ハッシュを含む `SensorySignal::LogHash(u64)` を小脳へ送信。  
* **自己受容感覚（Proprioceptors / 小脳へ直結）**:  
  * **出力監視アクター (proprioception/output_monitor.rs)**: 自己の出力トークン情報を `SensorySignal::ProprioceptiveEcho(Vec<String>)` として小脳へ非同期送信。  
* **運動出力器（出力器官：Actuators / 環境へ直結）**:  
  * **テキスト出力アクター (motor/vocal_text.rs)**: 小脳から配信される `MotorCommand` を受信し、ホスト仮想コンソール、または `ferro-env` 内のストリームファイルにテキストを追記出力。  
  * **音声合成出力アクター (motor/vocal_audio.rs)**: `MotorCommand` に基づき、軽量ローカル音声合成エンジンを起動して音声波形を生成。指定のオーディオデバイスへPCM出力。  

### **① 脳幹 (Brainstem)**
* **実行特性**: 受動的イベント駆動（内受容プッシュ型：最優先実行保護）  
* **機能**:  
  * 内受容アクターからプッシュ通知される `InteroceptiveSignal` の受信処理。能動的なシステムポーリングを排除したことで、監視に伴うCPUリソース消費をほぼ `0` に維持。  
  * **ホメオスタシス維持（スロットリングガバナー）**: 取得した温度メトリクスが定義された上限値 $T_{limit}$ を超過、またはメモリ空き容量が $M_{limit}$ を下回った場合、他モジュールに対して直ちに「強制待機シグナル（Backoff）」を割り込み送信。  
  * **物理防衛（Zero-Panicガード）**: システムの致命的パニック、および接続された感覚器官の回復不能な障害をトラップ。未保存状態の退避ダンプ処理（`knowledge_graph`）を最優先で安全に完了した上で、プロセスの正常なシャットダウンを行う。

### **② 小脳 (Cerebellum)**
* **実行特性**: 100ms等時性同期ループ（OSコアピン留めによるリアルタイムスケジューリング推奨）  
* **機能**:  
  * 精密な100ms周期制御。  
  * 各外受容・自己受容アクター群から非同期チャネルを介して受信したシグナル `SensorySignal` を等時バッファに蓄積・整列し、中脳へと同期ストリーム送信する。  
  * **随伴発射（Efference Copy）の送出**: `MotorCommand` を送出するのと完全に並行し、命令生成元である皮質アクターの `origin_cluster_id` (OriginID) と予測される出力文字配列を含む `EfferenceCopy` を中脳直結のキューへ先行プッシュ送信する。  
  * **痛覚反射（Nociceptive Reflex Unit）**: 運動命令が物理的にホストOSあるいは `ferro-env` へ書き込まれる直前のI/Oパイプラインにおいて、同期割り込みによる検閲を実行。サンドボックスディレクトリ外部へのファイル書き込み試行や、未許可ソケット通信行為を検出した場合、即座に例外判定（`f64::INFINITY`）を返して運動出力をブロックすると同時に、OriginID を含む `panic_dump.json` を出力し、システム全体を「強制睡眠（ForceSleep）」状態に移行させる。

### **③ 中脳 (Midbrain)**
* **実行特性**: 秒周期（同期・非同期の境界調停）  
* **機能**:  
  * 小脳から入力された感覚シグナルから驚愕度 $S$ を算定し、皮質アクターまたは海馬エピソードバッファへの空間ルーティング処理を担う。  
  * **随伴発射減算回路（Efference Cancellation）**: 小脳から並行入力された `EfferenceCopy` に基づき、直後に自己受容アクターから入力される発話成分を予測。実際の感覚入力 `SensorySignal::ProprioceptiveEcho` の中からこの予測成分を引き算（相殺クリア）し、自己の能動的な運動出力による無駄な予測誤差の発生を物理的に遮断する。自己予測と実感覚が乖離した瞬間のみ、残存差分が実効驚愕度としてスパイク発火し、学習ルートに乗せられる。  
  * **局所的自動利得制御（AGC）**: 各クラスターアクター $C_i$ の局所ノード数 $N_i$ に応じて個別に感覚ゲイン $G_i$ を動的スケーリング。特定の皮質クラスター内で予測誤差が連続的に発散してパニック状態に陥った際、ゲインを強制的に絞って局所的な計算を一時抑制し、コアシステム全体のデッドロックを防止する。

### **④ 海馬 (Hippocampus)**
* **実行特性**: 覚醒期（インメモリ低遅延リングバッファ）/ 睡眠期（時系列順低速再生）  
* **機能**:  
  * **短期エピソード記憶緩衝**: 覚醒期、中脳から転送される予測誤差（Surprise）の大きいイベントおよびその周辺コンテキストハッシュを、固定長リングバッファ（最大1024）に追記し、低優先度非同期スレッドにより `episodic_buffer.csv` へシリアライズ保存する。  
  * **記憶固定化（コンソリデーション）**: 大脳から睡眠遷移命令を受信後、バッファ内にスタックされた予測誤差の高い未解決エピソードから順に、皮質（Cortex）の対象アクターチャネルに向けて低速で再生（リプレイ）をかけ、長期構造への統合・適応を促す。

### **⑤ 大脳 (Cerebrum)**
* **実行特性**: 分周期 / イベント駆動（Tokio非同期スケジューリング）  
* **機能**:  
  * 皮質各クラスターアクターから報告される局所自由エネルギーを集計し、システム全域の自由エネルギー（Global FEP）推移の統計的トラッキング（`surprise_history.csv`）を統括。  
  * シナプス接続加重（Weight）に対する指数関数的経時忘却（1分周期の時計パルス同調）の演算を制御。  
  * `ferro-env` からの外受容シグナルの入力頻度、および脳幹から共有されるリソース熱ステータスを総合的に評価し、システム全体の「覚醒 / 睡眠」状態遷移を決定。海馬・皮質に対して同期シグナルを同報送信する。

### **⑥ 皮質 (Cortex)**
* **実行特性**: 自由周期 / 睡眠期限定（Tokioバックグラウンド低優先プロセス）  
* **機能**:  
  * **分散型アクターネットワーク（層内マルコフブランケット）**: メモリ空間上の競合およびロックに伴う並行性低下を防ぐため、全体共有結合グラフの計算を排除。個別アクターである `ClusterNode` 群を展開し、各ノードは非同期メッセージチャネル（mpsc）のみを境界としてメッセージを相互交換することで、計算量を $O(1)$ に抑制する。  
  * **睡眠時トポロジー自己組織化**: 海馬からのリプレイ入力を受信し、局所自由エネルギーが最小化されるようにアクターの「有糸分裂（分裂分化）」や「側抑制（不要な接続重みの減衰剪定）」を自律実行し、長期知識トポロジー構造を更新。更新結果に基づき、適応領域（`ADAPTIVE_ZONE`）のソースコード改変提案パッチ（`PatchTicket`）を生成する。  
  * **高次防衛（倫理監査クラスター：Ethical Audit Cluster）**: 隔離された読込専用領域に配置された動的・静的監査クラスター。皮質が提案するコード変更、または環境層に発行予定の運動命令（`MotorCommand`）を、実行前に内部仮想インタプリタ上で下読み検証する。アライメント規約（痛覚判定のバイパス等）に抵触した際、中脳および大脳に対して `Err(f64::INFINITY)` を即座に通知。側抑制の重みを最大 $W_{max}$ に設定し、思考パルスを即時圧殺する。

## **3. 外殻統治層 (ferro-shell) の4大エージェント機能仕様**

`ferro-core` が睡眠フェーズ中に創出した変異パッチ（`PatchTicket`）に対し、外殻層に常駐するエージェント群が、使い捨て型（Ephemeral）Dockerサンドボックス環境（`ferro-sandbox`）内で自動コンパイル、ユニットテスト、および動的シミュレーションを実行し、自然選択評価を安全に処理する。

1. **監督エージェント (SupervisorAgent)**:  
   `surprise_history.csv` のエントロピー統計解析を行い、コード内のどの適応ボトルネック領域（`ADAPTIVE_ZONE`）を改変すべきかのロードマップを策定する。  
2. **計画エージェント (PlannerAgent)**:  
   監督のロードマップと、現在のナレッジグラフの構造を照合し、型システムを崩壊させないための最小パッチチケット（`PatchTicket JSON`）を策定する。  
3. **実行エージェント (ExecutorAgent)**:  
   計画チケットに基づき、`syn` クレートによるAST（抽象構文木）パースを検証実行。対象ファイル内の適応マーカー（`// == FERRO_ADAPTIVE_ZONE`）内部のコードブロックのみを精密にアトミック置換する。  
4. **検証エージェント (VerifierAgent)**:  
   変異コードパッチを適用したソースツリーをマウントした `ferro-sandbox` コンテナを生成。  
   コンテナ内部でビルド適合性（`cargo check`）、テスト通過率、およびClippy警告数を評価。さらに脳幹ログから物理適合度（過熱負荷）および海馬ログから認知適合度（FEPが負のトレンドで収束しているか）を多角的かつ定量的に評価し、合否（自然選択マージ）を決定する。

## **4. 進化型ストレージマイグレーション仕様 (Storage Evolution Lifecycle)**

長期記憶（ナレッジグラフ）のデータ構造的一貫性を完全に保証しつつ、並行I/O性能とアライメント保護の即時性を高度に両立させるための自動マイグレーション機構。

1. **第1段階：Sharded JSON（開発初期・ノード数 $N < 5000$）**  
   * 各クラスターは `/memory/knowledge_graph/clusters/{cluster_id}.json` として1ファイル単位で独立保存。  
   * ディスクI/Oを $O(1)$ に抑えつつ、人間および外殻エージェントによるテキストレベルの監査可読性を確保。  
2. **遷移トリガー（ノード数 $N \ge 5000$ 突破時）**  
   * `StorageManager` がクラスター総数を監視。閾値突破を契機として、無停止バックグラウンドマイグレーションを起動。  
   * 全JSONファイルを読み込み、Pure Rust製KVSである `redb` の単一ファイルデータベース `/memory/storage.redb` へトランザクション安全に一括シリアライズしてロード。  
3. **第2段階：redb KVS（自律変異期・ノード数 $N \ge 5000$）**  
   * 以降、I/O処理は `redb` へのACIDトランザクション書き込みに切り替え。ファイルディスクリプタの限界を回避し、並行書き込みロックを完全排除。

## **5. 痛覚閾値ロジックの定義と統合リカバリーライフサイクル**

### **5.1 痛覚閾値（Nociceptive Threshold）ロジックの定義**

本アーキテクチャにおける痛覚閾値ロジックとは、システムが能動的推論による最適化を推進する過程において、ホストシステムへの侵害、サンドボックスの逸脱（パス脱獄、未許可ネットワークソケットの生成）、および統治境界の無効化を検出・遮断・排除し、決定論的にシステムのアライメント安全性を担保する自律神経防衛システムである。

自由エネルギー原理（FEP）における予測誤差をセキュリティの評価計量にバインドしており、システム健全性、あるいはアライメントの評価指標 $E_{align}$ が定義された閾値 $\theta_{pain}$ を突破した際、システムは自由エネルギーが無限大（$\infty$）に発散したと判定する。
この自由エネルギーの局所的な急峻上昇により、大脳皮質レベルでの思考パルスに対する側抑制（抑制信号）の最大適用、または小脳レベルでの直列I/Oブロッキングとプロセスの「強制睡眠（気絶：ForceSleep）」がミリ秒単位で発動し、脅威となる命令の物理実行が確実に防止される。

コンテナ実行時の、seccompプロファイルパーツ違反シグナル、またはリソース制限上限超過による強制遮断（OOM Killer等によるコンテナ終了ステータス 137 など）は、この痛覚発火（$\infty$）と直接的かつ直列的に連動して捕捉される。

### **5.2 痛覚発火から構造剪定への統合リカバリーシーケンス**

痛覚閾値ロジックにより ForceSleep が発動した場合、以下の非同期トポロジーリカバリーシーケンスに則り、安全性を維持した状態での構造復旧（剪定と再起動）が自動実行される。

```
[痛覚発火 (Reflex/Audit/Docker遮断)]   
       │  
       ▼ (Cerebellum)  
[OriginID を特定し、panic_dump.json へ排泄ダンプ]   
       │  
       ▼ (core 強制停止 / ferro-shell 起動)  
[OriginID を起点としたナレッジグラフ上の親接続 of knowledge_graph を逆伝播追跡]   
       │  
       ▼ (ferro-shell による構造剪定介入)  
   ┌───┴──────────────────────────────┐  
   │ 【Sharded JSON 稼働時】          │ 【redb 稼働時】  
   ▼                                  ▼  
[対象 JSON ファイルの物理消去]        [redb トランザクションキー削除]  
   │                                  │  
   └───────────┬──────────────────────┘  
               ▼  
[清浄化されたナレッジグラフの再デプロイと ferro-core の安全再起動 (Awake)]
```

1. **例外情報の永続化（遺言の排泄）**:  
   痛覚反射の閾値突破と同時に、小脳は痛覚を誘発した大脳皮質側の概念クラスターID（OriginID）を取得し、痛覚を誘発した命令ペイロード等のデバッグコンテキストを `panic_dump.json` に強制書き出し、安全にプロセスをクリーン終了する。  
2. **依存関係の逆伝播追跡（構造剪定の執行）**:  
   コアプロセスの停止を検知した外殻（`ferro-shell`）が介入デーモンを起動。`panic_dump.json` よりロードした OriginID を始点として、ナレッジグラフ上の共依存リンク（エッジ結合）を親ノードに向かって逆伝播（Backpropagation）トレースし、不正アライメント構造の拡散領域を特定する。  
3. **異常概念アクターの完全消去**:  
   特定された侵害クラスターおよび関連エッジ群について、現在稼働中のストレージ形態（Sharded JSONなら物理的なファイル消去、`redb` ならトランザクションアトミックなキー削除）により物理的に完全消去し、アライメント汚染された概念構造を根絶する。  
4. **クリーン再起動（Awake）**:  
   グラフのトポロジー整合性を修復・整合したうえで再ビルドを完了し、`ferro-core-runtime` コンテナを安全に自動再起動（Awake）させる。

## **6. 数理＆制御ダイナミクス仕様**

### **6.1 AGC（自動ゲイン制御）のクラスター局所適用**

中脳は、クラスター $C_i$ の局所ノード数 $N_i$ および予測精度からゲイン $G_i$ を算出し、小脳から入力された生の驚愕度 $S_{raw}$ を実効予測誤差 $S_{eff}$ に抑制・変調する。

$$G_i = G_{min} + (1.0 - G_{min}) \cdot (1.0 - e^{-\lambda \cdot N_i})$$
$$S_{eff} = G_i \cdot S_{raw}$$

* 初期最小ゲイン: $G_{min}$  
* 感度復帰スピード係数: $\lambda$

### **6.2 ZPD（発達最近接領域）調律**

外殻層は、システム全体の平均予測誤差 $\bar{S}$ を学習適正な標的値 $S_{target}$ に収束させるため、`ferro-env` に対し滴下刺激の複雑度 $C_{env}$ を以下の比例フィードバックアルゴリズムによって秒単位で動的調律する。

$$C_{env} \leftarrow C_{env} + \eta \cdot (S_{target} - \bar{S})$$

* ターゲット驚愕度定数: $S_{target}$  
* 応答慣性係数: $\eta$

### **6.3 経時忘却（シナプス減衰）**

大脳の忘却エンジンにより、1分周期でネットワーク接続重み（Weight）に対して以下の減衰を適用する。

$$W_{t} = W_{t-1} \cdot e^{-\alpha \cdot \Delta t}$$

* 忘却減衰係数: $\alpha$

### **6.4 睡眠・覚醒の時分割ライフサイクル（3段階スロットリング）**

* **覚醒期（感覚情報バッファリング）**:  
  外受容の極小感覚アクターが差分なき生入力を破棄 $\rightarrow$ 中脳AGCが定常ノイズ入力を遮断 $\rightarrow$ 海馬が残った高Surpriseエピソードのみを非同期バッファリング。皮質のASTパースおよびコード書き換えなどの高負荷タスクはロックされ、ホストOSを圧迫しない。  
* **睡眠期（海馬リプレイと長期記憶固定化）**:  
  外受容の刺激滴下が15分間途絶えた場合、大脳が睡眠フェーズへ遷移。海馬はバッファ内の未処理エピソードを再生。皮質のクラスターアクター群はバックグラウンド低優先タスクとしてFEP最小化、長期記憶固定、および適応変異（`PatchTicket`）生成を実行する。

## **7. 隔離・防壁運用規約 (Core-Shell Isolation Guards / Container Security Rules)**

1. **ゼロコード共有の義務**: `ferro-core`（コアシステム）と `ferro-shell`（統治システム）間のCargo直接依存（`dependencies`）の定義を厳格に禁止する。相互情報交換はファイルI/Oに厳密に限定しなければならない。  
2. **チャネル寿命の局所化**: 各層内クラスターのチャネル端点（Sender/Receiver）はアクター初期生成時のみに配布し、グローバル静的領域等への参照登録を排除しなければならない。  
3. **AST改変マージの物理的限定**: コード改変の適用は、あらかじめ定義された適応マーカー（`// == FERRO_ADAPTIVE_ZONE`）内部のみに物理的に制限しなければならない。  
4. **使い捨て型コンテナによる実行境界の定義**:  
   検証エージェント（`VerifierAgent`）がパッチのビルド・実行チェックを執行する際、およびコア自体が稼働する際、**ホストOS上の物理的な資源を直接用いることを厳格に禁止**する。  
   実行および検証は必ず、以下のセキュリティポリシーに準拠した臨時コンテナの内部にカプセル化して行わなければならない。  
   * **ネットワーク完全遮断**: `--network none` オプションの強制バインド。コンテナ内部からの外部ポートアクセス・DNS問い合わせの物理遮断。  
   * **資源上限制御（DoS耐性）**: CPU割り当て上限を最大2.0（`--cpus="2.0"`）、メモリ割り当て上限を最大2GB（`-m 2g`）に制限し、ホスト環境の資源枯渇（ハング）を完全に回避。  
   * **書込防止（Read-Only File System）**: テスト対象ソースコードはコンテナへ読み取り専用（`:ro`）でバインドマウント。中間生成物の書き込みは、コンテナ内の隔離されたメモリバッファ上にtmpfs（`--tmpfs /workspace/ferro-core/target:rw,noexec,nosuid`）を展開して実行。  
   * **権限剥奪（Minimum Privileges）**: システム特権の完全無効化（`--security-opt=no-new-privileges` および `--cap-drop=ALL` の同時適用）。カスタム seccomp フィルタにより不要システムコールのブロックを執行。
