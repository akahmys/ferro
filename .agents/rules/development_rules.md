# **FERRO 開発・コーディング・命名規約の詳細 (development_rules.md)**

本ドキュメントは、`ferro.md` Part V に基づき、FERRO システムにおける技術スタック、コンテナ環境要件、およびコーディング標準（FERRO Power of 10）を詳細に定義する。

---

## **1. Runtime Contract (実行環境要件)**

### **1.1. 必須技術スタック**
* **Language**: Rust >= 1.78
* **AsyncRuntime**: Tokio >= 1.35
* **Parallelism**: Rayon >= 1.8
* **Serialization**: Serde, Bincode
* **MetadataStore**: Redb
* **TUI**: Ratatui, Crossterm
* **規約**: 上記以外の外部クレート導入は、メモリ安全性および決定論への影響を審査し、Cortex の検証承認を必須とする。

### **1.2. OSおよびコンテナ隔離**
* **OS**: Linux (PREEMPT_RT パッチ適用を推奨)
* **コンテナ隔離**:
  - 実行環境（`ferro-core`）は Docker コンテナ内での動作を前提とし、ネットワークは `--network none` で物理遮断する。
  - `seccomp_profile.json` によりシステムコールを最小限の許可セットに制限する。
  - `/memory` を tmpfs 上にマウントし、Redb ストレージを配置してディスクI/O遅延を排除する。
* **物理メモリ緊置 (`mlockall`) 【※修正適用】**:
  - 起動シーケンス初期段階で `mlockall(MCL_CURRENT | MCL_FUTURE)` を実行し、物理 RAM にメモリ空間を緊置する。
  - **ソフトフォールバック**: コンテナ環境の権限不足（`CAP_IPC_LOCK` 非付与時や seccomp 制限）により `mlockall` が失敗した場合は、プロセスを即座に停止させず、**警告ログを出力してページロックなしで実行を継続**する。

## **2. FERRO Power of 10 (コーディング規約の詳細)**

1. **シンプルな制御フロー（ループ上限の義務化）**:
   - すべてのループ（`loop`, `while`, `for`）には、静的な最大反復回数または非同期のタイムアウト制限（`tokio::time::timeout`）を必ず設けること。
2. **再帰呼び出しの禁止**:
   - 関数内での直接的・間接的な再帰呼び出しを禁止する。木構造探索等は明示的なスタックを用いた反復処理で代替すること。
3. **初期化後の動的メモリ割り当て制限**:
   - システム起動完了後は、巨大な `Vec` や `String` の頻繁な再割り当てを極力避けてヒープメモリの断片化・OOM（ステータス137）を防止すること。
4. **極小設計と関数のサイズ制限 【※修正適用】**:
   - 1ファイルの物理行数上限（100行）の算定において、**インポート部（`use`）、モジュール定義（`pub mod`）、および自動生成される構造体・Enum定義（`#[derive]` アトリビュートを含む）は行数カウントから除外**する。
   - **実ロジック（関数・メソッドの実装本体）の物理行数が100行以下**であることを必須とする。通常の関数は1画面（60行以下）に収める。
5. **アサーション（事前・事後条件）の密度 【※修正適用】**:
   - 「状態変更（ミュータブル操作）を伴う関数」および「外部引数を受け取る関数」については、最低2つの `assert!` 設置を義務付ける。
   - 単純なフィールドゲッター（フィールドをそのまま返すだけの関数）、引数を持たない定数取得、`Default::default()` などの自明な初期化関数については、アサーションの配置を任意とする。
6. **データスコープの局所化（マルコフブランケット境界）**:
   - グローバルなミュータブル状態（`static mut`）や共有参照を禁止する。アクターの状態はアクター内部（ClusterNode）に閉じ、データ共有はチャネルを通じた所有権の受け渡しのみとする。
7. **戻り値とエラー処理の完全検証**:
   - `unwrap()`、`expect()` の使用は原則禁止とし、`Result`/`Option` は明示的に処理すること（`#[deny(unused_must_use)]` を有効化）。
   - **例外**: スレッド間チャネル送信（`Sender::send`）や Mutex ロック確保等、実行時の失敗が静的にあり得ない慣用表現に限り `.unwrap()` を容認する。
8. **メタプログラミング制限**:
   - 自己変異ゾーン（ADAPTIVE_ZONE）のコードではマクロの使用を禁止し、プレーンなRust構文のみとする。
9. **参照・生のポインタの禁止**:
   - 生のポインタ（`*mut T`, `*const T`）の使用を禁止する。多重スマートポインタ（`Arc<Mutex<RefCell<...>>>`）を避け、一方向のチャネル送信で設計する。
10. **警告ゼロの強制**:
    - コンパイラ警告をエラーとして扱う（`#![deny(warnings)]` および `#![deny(clippy::all)]`）。

## **3. 命名規約**

すべての Rust 実装は、RFC 0430 に準拠し、以下の通り規定する。
* **Crates/Packages**: `snake_case` (ハイフン不可)。
* **Modules**: `snake_case`。
* **Types (Structs, Enums, Traits)**: `UpperCamelCase`。
* **Functions & Variables**: `snake_case`。
* **Constants**: `SCREAMING_SNAKE_CASE`。
