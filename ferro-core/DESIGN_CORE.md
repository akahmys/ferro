# **FERRO コアロジック層の設計・アルゴリズム詳細 (DESIGN_CORE.md)**

本ドキュメントは、`ferro.md` Part II（コアアーキテクチャ契約）および Part III（アルゴリズム契約）に基づき、`ferro-core` における各機能モジュールの要件と数理計算プロセスを定義する。

---

## **1. コアアーキテクチャ (Core Architecture)**

### **1.1. DynamicClusterNode (推論最小単位)**
* **責任**: 局所事後分布を保持する唯一のオブジェクトであり、予測誤差の時系列履歴をカプセル化する。
* **不変条件**: 内部パラメータは浮動小数点演算精度範囲（f64）内でクランプする。前回の状態と更新後の状態がアトミックに入れ替わるトランザクション（`with_mut_node`）を保証する。

### **1.2. Brainstem (脳幹 - 生存・代謝監視)**
* **責任**: CPU使用率、メモリ占有率、推論サイクルの遅延を常時監視し、物理的生存を保証する。
* **不変条件**: メモリ 90% 超過やフリーズ検知時、直ちに `ForceSleep` を発動し、FEP-State をフラッシュして安全にクリーン終了させる。

### **1.3. Cerebellum (小脳 - リズム制御・等時同期)**
* **責任**: 100ms周期の同期信号を生成し、全ノードを強制同期させるマスタークロック。
* **不変条件**: サイクル遅延が 10ms を超えた場合、即座に当該計算を破棄し、Cortex へオーバーロード警告を発し、Shell へ緊急減速フラグを通知する。

### **1.4. Midbrain (中脳 - Surprise ルーティング・自己消去)**
* **責任**: 感覚入力の自動利得制御（AGC）と、驚き（Surprise）に基づく Cortex への優先ルーティング。
* **随伴発射相殺**: 自己発話（運動出力）時、小脳から受ける随伴発射と自己受容アクターから戻るエコーを時間窓内で照合し、Surprise レベルを 0 に相殺（Efference Cancellation）して耳アクターへ `SensoryMuteCommand` を送出する。

### **1.5. Hippocampus (海馬 - 記憶バッファ)**
* **責任**: 覚醒期のエピソード的経験（入力、出力、Surprise）を `episodic_buffer.csv` に保存し、睡眠期において Cortex に対しリプレイ（オフライン再構成）を指揮する。

### **1.6. Cerebrum (大脳 - 全域CSR管理)**
* **責任**: 全域の変分自由エネルギーの集計、および重み行列の CSR (Compressed Sparse Row) 形式での管理・最適化（不要な接続の動的剪定）。

### **1.7. Cortex (大脳皮質 - 倫理監査・クラスター適応)**
* **責任**: 予測誤差上昇に伴うクラスターの自律的分裂（Mitosis）と側抑制、仮想ATP代謝による餓死・剪定、およびアライメントスコア $A_s$ のリアルタイム監査。

## **2. アルゴリズム契約 (Algorithm Contracts)**

### **2.1. FEP (Free Energy Principle) 計算**
* **局所自由エネルギー**: $F_i = \alpha E_i + \beta V_i + \gamma C_i$ を数値演算順序固定で逐次計算する（丸め誤差の排除）。

### **2.2. Expected Free Energy (EFE) 推定**
* **期待自由エネルギー**: Pragmatic Value と Epistemic Gain（情報利得）の和 $G(\pi)$ を最小化する方策を選定し、Active Inference を保証する。

### **2.3. SPMV 疎行列ベクトル積 ＆ 決定論的並列 Map/Reduce 【※修正適用】**
* **結合重み行列演算**:
  - 重み行列 $W$ は CSR 形式で管理し、インデックス昇順で CPU キャッシュヒット率を最大化する。
  - Lipschitz 境界（$\sum |w_{ij}| \leq 3.6$）の遵守を絶対条件とする。
* **決定論的並列処理モデル**:
  - **Map フェーズ (並列実行)**: 副作用のない独立した個別ノードの勾配・予測誤差の計算は、Rayon を用いて並列に処理する。
  - **Reduce フェーズ (順序固定直列)**: 全体の自由エネルギー集計や重みの更新値の積算は、必ず**ノードIDの昇順（ソート順）**に順序を固定し、同一スレッド上で直列に演算する。
  - これにより、スレッドスケジューリングによる浮動小数点の丸め順序のゆらぎを完全に排除し、ビット単位の決定性を保証する。

### **2.4. CSR Cache 管理**
* Hippocampus の予測に基づき、次サイクルで活性化する接続を優先的にプリフェッチする。

### **2.5. Sleep Consolidation (睡眠時コンソリデーション)**
* 睡眠期においてエピソードを再現し、全域目的関数 $J = \sum \text{EFE}_i + \gamma(1 - A_s)$ を最小化するよう重みを最適化する。最適化前後のエネルギー差 $\Delta J < 0$ の場合のみ変更をアトミックに確定する。

### **2.6. Undo Transaction (トランザクション復元)**
* `with_mut_node()` 中に数理制約違反が発生した場合、即座に遷移を破棄し、トランザクション開始前の状態をアトミックに復元して `panic_dump.json` に詳細を出力する。
